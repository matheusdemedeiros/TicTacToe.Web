# Instruções para testes unitários

## Framework

- **xUnit** como framework de testes
- **FluentAssertions** para asserções legíveis

## Organização

- Testes organizados por módulo, espelhando a estrutura do Domain:
  - `MatchModule/TicMatchTest.cs`
  - `MatchModule/TicPlayerTest.cs`
  - `MatchModule/TicBoardTest.cs`
  - `MatchModule/TicBoardCellTest.cs`
  - `MatchModule/TicScoreTest.cs`

## Convenções

- Nomes de teste seguem o padrão: `MetodoOuCenario_Condicao_ResultadoEsperado`
- Testar validações de domínio verificando que `DomainException` é lançada com a mensagem correta.
- Testar caminhos felizes e cenários de erro.
- Sem mocks para testes de domínio — testar entidades diretamente.
