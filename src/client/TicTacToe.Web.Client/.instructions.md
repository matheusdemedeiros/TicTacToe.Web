# Angular Frontend Instructions

## Stack
- Angular 19 with standalone components
- @microsoft/signalr for WebSocket communication
- RxJS for reactive data flow
- SCSS for styles

## Project Structure

```
src/app/
+-- components/
|   +-- home/                     (lobby: create player, create/join match)
|   |   +-- shared/
|   |       +-- models/           (TypeScript interfaces and enums)
|   |       +-- services/         (MatchService, PlayerService - HTTP calls)
|   +-- match/                    (gameplay: board, cells, match logic)
|       +-- shared/
|       |   +-- models/           (TicMatch, TicBoardCell, TicScore interfaces)
|       |   +-- services/         (TicMatchHubService - SignalR, hub message models)
|       +-- tic-match/            (match component - orchestrates game)
|       +-- tic-board/            (board component - renders 3x3 grid)
|       +-- tic-board-cell/       (cell component - individual cell)
+-- core/                         (cross-cutting: NotificationService)
```

## Conventions
- Use standalone components (no NgModules)
- Use `inject()` function for dependency injection
- Models are TypeScript interfaces (not classes)
- Services are `providedIn: 'root'`
- Hub service wraps SignalR callbacks in RxJS `Observable`
- REST services use Angular `HttpClient`

## Backend Communication
- REST base URL: `environment.apiUrl + 'api'`
- SignalR Hub URL: `environment.apiUrl + 'Ticmatchhub'`
- Backend uses Newtonsoft.Json: property names come in **PascalCase from the server**. Watch for casing mismatches in TypeScript interfaces.

## Shared Enums (must match backend values)
- TicMatchState: NOT_STARTED=0, IN_PROGRESS=1, FINISHED=2
- PlayModeTypes: PlayerVsPlayer=0, PlayerVsComputer=1
- TicBoardCellState: BLANK=0, MARKED=1
