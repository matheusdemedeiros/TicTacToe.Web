# Angular Frontend Instructions

## Stack
- Angular 19 with standalone components
- @microsoft/signalr for WebSocket communication
- RxJS for reactive data flow
- SCSS for styles

## Project Structure

```
src/app/
+-- components/
|   +-- home/                     (lobby: create player, create/join match)
|   |   +-- shared/
|   |       +-- models/           (TypeScript interfaces and enums)
|   |       +-- services/         (MatchService, PlayerService - HTTP calls)
|   +-- match/                    (gameplay: board, cells, match logic)
|       +-- shared/
|       |   +-- models/           (TicMatch, TicBoardCell, TicScore interfaces)
|       |   +-- services/         (TicMatchHubService - SignalR, hub message models)
|       +-- tic-match/            (match component - orchestrates game)
|       +-- tic-board/            (board component - renders 3x3 grid)
|       +-- tic-board-cell/       (cell component - individual cell)
+-- core/                         (cross-cutting services)
|       +-- notification.service  (toastr wrapper: success, error, warning, info)
|       +-- game-session.service  (localStorage: matchId + playerId for reconnection)
|       +-- http-error.interceptor (global HTTP error -> toastr)
|       +-- api-error-response.model (backend error interface)
```

## Conventions
- Use standalone components (no NgModules)
- Use `inject()` function for dependency injection
- Models are TypeScript interfaces (not classes)
- Services are `providedIn: 'root'`
- Hub service wraps SignalR callbacks in RxJS `Observable`
- REST services use Angular `HttpClient`

## Backend Communication
- REST base URL: `environment.apiUrl + 'api'`
- SignalR Hub URL: `environment.apiUrl + 'Ticmatchhub'`
- Backend uses Newtonsoft.Json: property names come in **PascalCase from the server**. Watch for casing mismatches in TypeScript interfaces.

## Error Handling

### HTTP Error Interceptor (`httpErrorInterceptor`)
- Registered globally in `app.config.ts` via `withInterceptors([httpErrorInterceptor])`
- Intercepts ALL HTTP error responses automatically
- Extracts `message` from the backend's `ApiErrorResponse` JSON and shows it via toastr
- **Do NOT add `error:` callbacks in `subscribe()`** for API calls — the interceptor handles display
- If connection is lost (status 0), shows 'Unable to connect to the server.'

### Notification Service (`NotificationService`)
- `showSuccess(message, title)` - Green toast for successful operations
- `showError(message, title)` - Red toast for errors
- `showWarning(message, title)` - Yellow toast for warnings
- `showInfo(message, title)` - Blue toast for informational messages
- Configured in `app.config.ts`: 4s timeout, top-right position, progress bar, prevent duplicates

### Backend Error Response Format
`json
{ ""message"": ""Player with this nickname already exists."", ""errorCode"": ""DOMAIN_ERROR"", ""statusCode"": 400 }
`
TypeScript interface: `IApiErrorResponse` in `core/api-error-response.model.ts`

### SignalR Error Handling
- Errors from Hub `invoke()` calls are caught via `.catch()` in `TicMatchHubService`
- Connection loss triggers automatic reconnection with a warning toast
- Hub method errors show the server error message via toastr

## Unified Game State

Both SignalR events (TicPlayerJoined, TicPlayerMadeMove) use `ITicMatchStateResponse`:
- Contains all game state: board, players, turn, finish status, winner/tie info
- Frontend maps it to `TicMatch` interface via `updateMatchState()` in TicMatchComponent
- Defined in `match/shared/services/hub-messages.model.ts`

## Board Locking

- `isBoardLocked` getter combines: not my turn + game over + match not in progress
- Passed as `[disabled]` input to TicBoardComponent and TicBoardCellComponent
- Cell buttons have `[disabled]` attribute + visual styling via `[ngClass]`

## Game Session (Reconnection)

- `GameSessionService` saves matchId + playerId to localStorage on match creation/join
- TicMatchComponent reads from localStorage if query params are missing
- Enables browser refresh / tab close recovery
- Session cleared when game finishes

## Shared Enums (must match backend values)
- TicMatchState: NOT_STARTED=0, IN_PROGRESS=1, FINISHED=2
- PlayModeTypes: PlayerVsPlayer=0, PlayerVsComputer=1
- TicBoardCellState: BLANK=0, MARKED=1
