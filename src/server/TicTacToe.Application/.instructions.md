# Instruções para a camada Application

## Padrão CQRS com MediatR

Cada Use Case é composto por 3 arquivos dentro de uma pasta com o nome da ação:

```
UseCases/Match/{NomeAção}/
  ├── {NomeAção}Command.cs   (ou Query para leitura)
  ├── {NomeAção}Handler.cs
  └── {NomeAção}Response.cs
```

## Regras

- Commands implementam `IRequest<TResponse>`.
- Handlers implementam `IRequestHandler<TCommand, TResponse>`.
- Handlers recebem repositórios e `IUnitOfWork` via injeção de dependência.
- Não coloque lógica de domínio nos Handlers — delegue para as entidades.
- Responses são classes simples (DTOs) sem lógica.

## Tratamento de erros nos Handlers

- **Nunca retorne `null`** para indicar erro. Lance `DomainException` com mensagem descritiva em inglês.
- **Nunca lance `Exception` genérica.** Use sempre `DomainException` (de `TicTacToe.Domain.SharedModule.Exceptions`).
- O `GlobalExceptionHandlerMiddleware` na camada WebAPI cuida de transformar a exceção em resposta HTTP adequada.
- Os Controllers não fazem validação de `null` — confiam no Handler para lançar exceção quando necessário.

```csharp
// ✅ Correto
if (!playerExists)
    throw new DomainException("Player not found.");

// ❌ Errado
if (!playerExists)
    return null;

// ❌ Errado
throw new Exception("Player not found.");
```

## Use Cases existentes

| Use Case | Tipo | Descrição |
|----------|------|-----------|
| CreateMatch | Command | Cria uma nova partida |
| CreatePlayer | Command | Get-or-create: retorna jogador existente se nickname já existe |
| AddPlayer | Command | Adiciona jogador a uma partida |
| JoinMatch | Command | Jogador entra na partida (SignalR), inicia se 2 jogadores, suporta rejoin |
| MakeMove | Command | Realiza uma jogada |
| RetrieveMatchById | Query | Busca partida por ID |

## Response unificada: TicMatchStateResponse

`JoinMatch` e `MakeMove` retornam `TicMatchStateResponse` (em `UseCases/Match/Shared/`).
Usar o factory method `TicMatchStateResponse.FromMatch(match)` para construir a response.
Nunca criar responses manuais com campos avulsos — usar sempre o factory method.

## Registro de serviços

Os serviços da camada Application são registrados em `Services/ServiceExtensions.cs` via `AddTicApplication()`.
