# Instruções para a camada Domain

## Regras

- Entidades **sempre** herdam de `BaseEntity`.
- Use `private set` em propriedades para proteger invariantes do domínio.
- Validações de negócio devem lançar `DomainException` com mensagem descritiva em inglês.
- O Domain **não** referencia nenhum pacote externo de infraestrutura (EF Core, MediatR, etc.).
- Interfaces de repositório ficam em `Domain/Interfaces/` organizadas por módulo.
- Enums de domínio ficam junto das entidades no módulo correspondente.

## Entidades existentes (módulo Match)

- `TicMatch` — Partida (Players, Board, State, Score, CurrentPlayer, PlayMode)
- `TicPlayer` — Jogador (Name, NickName, Symbol)
- `TicBoard` — Tabuleiro 3x3, serializado como JSON para persistência
- `TicBoardCell` — Célula do tabuleiro
- `TicScore` — Placar (WinningPlayer, WinningSymbol, Tie)
- `TicMatchState` — Enum de estado da partida
- `PlayModeType` — Enum de modo de jogo

## Padrão para novas entidades

```csharp
public class NomeEntidade : BaseEntity
{
    // Propriedades com private set
    public string Propriedade { get; private set; }

    // Construtor protegido para EF Core
    protected NomeEntidade() : base() { }

    // Construtor público com validação
    public NomeEntidade(string propriedade) : base()
    {
        if (string.IsNullOrEmpty(propriedade))
            throw new DomainException("Mensagem descritiva.");

        Propriedade = propriedade;
    }
}
```
